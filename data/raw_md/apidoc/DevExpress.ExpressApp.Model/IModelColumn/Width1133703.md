### ASP.NET Core Blazor UI

Use the `Width` property to set a grid column width. This property value is passed to the actual ASP.NET Core Blazor Grid Control’s [DxGridColumn.Width](xref:DevExpress.Blazor.DxGridColumn.Width) property in pixels.

When this property is not specified in the Model Editor, grid columns evenly occupy the whole grid width. When this property is set for a part of grid columns, other columns are resized according to the available space. When it is set for all columns, the grid control either displays a horizontal scrollbar or an empty space. This also happens after resizing columns in the Columns node designer ([](xref:113679)) and at runtime.

### WinForms UI

Column width in `XtraGrid` is controlled by the [GridOptionsView.ColumnAutoWidth](xref:DevExpress.XtraGrid.Views.Grid.GridOptionsView.ColumnAutoWidth) property, so actual column width can differ from the `Width` property value. If a ListView has many columns, their captions and content may be truncated. In this case, set the [GridOptionsView.ColumnAutoWidth](xref:DevExpress.XtraGrid.Views.Grid.GridOptionsView.ColumnAutoWidth) property to `false` to show a horizontal scrollbar. For information on how to access the Grid Control settings, refer to the following article: [](xref:402154).
