When modifying this property value via the [Model Editor](xref:112582), note the specific meaning if the **Index** is zero. Initially, the **Index** is zero, and it means that the property value has not been changed, 
the node order is not customized, and nodes are arranged by default. However, if you set **Index**, for example, to "1", then subsequently set it to "0", then it will be recognized as a user difference (saved in the XAFML file), and will actually influence the node order.

> [!NOTE]
> You can rearrange the child nodes of a particular node via the ALT+UP ARROW and ALT+DOWN ARROW shortcuts in the Model Editor. Their indexes will be updated automatically to correspond the rearrangements
