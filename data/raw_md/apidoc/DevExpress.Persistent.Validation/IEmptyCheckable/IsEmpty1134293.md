Support the [](xref:DevExpress.Persistent.Validation.IEmptyCheckable) interface and implement the **IsEmpty** method, when it is required to validate properties of your custom type via the **RuleRequiredField** attribute. Below is the snippet from the [Business Class Library's](xref:112571) **FileData** class, illustrating the implementation of this property:

# [C# (EF Core)](#tab/tabid-csharp-ef)

```csharp
public class FileData :  BaseObject, IFileData, IEmptyCheckable {
    // ...
    public virtual string FileName { get; set; }
    // ...
    [Browsable(false)]
    public bool IsEmpty {
        get { return string.IsNullOrEmpty(FileName); }
    }
}

// Make sure that you use options.UseChangeTrackingProxies() in your DbContext settings.
```

# [C# (XPO)](#tab/tabid-csharp-xpo)

```csharp
public class FileData : BaseObject, IFileData, IEmptyCheckable {
    // ...
    public string FileName {
        get { return fileName; }
        set {
            SetPropertyValue<string>(nameof(FileName), ref fileName, value);
        }
    }
    // ...
    [NonPersistent, MemberDesignTimeVisibility(false)]
    public bool IsEmpty {
        get { return string.IsNullOrEmpty(FileName); }
    }
}
```
***

With the **IEmptyCheckable** interface supported, the property of the **FileData** type can be validated in the following manner:

# [C# (EF Core)](#tab/tabid-csharp-ef)

```csharp
public class Resume : BaseObject {
     // ...
    [RuleRequiredField("RuleRequiredField for Resume.FileData", 
        DefaultContexts.Save, ,"The file must be specified"]
    public virtual FileData File { get; set; }
    // ...
}

// Make sure that you use options.UseChangeTrackingProxies() in your DbContext settings.
```

# [C# (XPO)](#tab/tabid-csharp-xpo)

```csharp
public class Resume : BaseObject {
     // ...
    [RuleRequiredField("RuleRequiredField for Resume.FileData", 
        DefaultContexts.Save, ,"The file must be specified"]
    public FileData File {
        // ...
    }
    // ...
}
```
