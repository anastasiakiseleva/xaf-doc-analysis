Apply this attribute to a property to define a validation rule that will check whether this property value is specified. Use the common parameters that are inherited from the [](xref:DevExpress.Persistent.Validation.RuleBaseAttribute) class.

# [C# (EF Core)](#tab/tabid-csharp-ef)

```csharp
[DefaultClassOptions]
[System.ComponentModel.DefaultProperty(nameof(Title))]
public class Position : BaseObject {
   [RuleRequiredField("RuleRequiredField for Position.Title",DefaultContexts.Save,
      "A title must be specified")]
   public virtual string Title { get; set; }
}

// Make sure that you use options.UseChangeTrackingProxies() in your DbContext settings.
```

# [C# (XPO)](#tab/tabid-csharp-xpo)

```csharp
[DefaultClassOptions]
[System.ComponentModel.DefaultProperty(nameof(Title))]
public class Position : BaseObject {
   //...
   private string title;
   [RuleRequiredField("RuleRequiredField for Position.Title",DefaultContexts.Save,
      "A title must be specified")]
   public string Title {
      get { return title; }
      set { SetPropertyValue(nameof(Title), ref title, value); }
   }
}
```
***

When a property is decorated with a `RuleRequiredFieldAttribute`, the property value is not valid in the following cases:

- The property value is `null`. 
- The property is string, and its value is an empty string.
- The property is of type [](xref:System.DateTime), and its value is `DateTime.MinValue`
- The property type implements the [](xref:DevExpress.Persistent.Validation.IEmptyCheckable) interface, and [IEmptyCheckable.IsEmpty](xref:DevExpress.Persistent.Validation.IEmptyCheckable.IsEmpty) is `true`.
- The property type implements the [](xref:System.Collections.IEnumerable) interface, and the property value is an empty collection.

If any of these clauses is true, the property value is not valid. The clause priority corresponds to the order in the list above. You can handle the [RuleSet.CustomIsEmptyValue](xref:DevExpress.Persistent.Validation.RuleSet.CustomIsEmptyValue) event to override this behavior.

> [!NOTE]
> If the target property is decorated with the [](xref:DevExpress.Xpo.ValueConverterAttribute), [](xref:DevExpress.Xpo.Metadata.ValueConverter) is ignored during validation. For example, if the database value is `null` but the actual value is not, the property value is valid.

### Requirements

XAF requires that the properties decorated with the [RuleRequiredFieldAttribute](xref:DevExpress.Persistent.Validation.RuleRequiredFieldAttribute) attribute must be of nullable or reference type. Otherwise, the validation rule does not work.

In most cases, `RuleRequiredFieldAttribute` is used when you need to ensure that a property has a value. However, some scenarios require [](xref:DevExpress.Persistent.Validation.RuleCriteriaAttribute) or [](xref:DevExpress.Persistent.Validation.RuleValueComparisonAttribute). For instance, `RuleRequiredFieldAttribute` does not validate enumerations, as they are value types. You can use the following technique to validate an enumeration property value:

# [C# (EF Core)](#tab/tabid-csharp-ef)

```csharp
[RuleValueComparison(
    null, DefaultContexts.Save, ValueComparisonType.NotEquals, Priority.Unspecified)]
public virtual Priority Priority { get; set; }

// ...
public enum Priority {
    Unspecified,
    Low,
    Normal,
    High
}

// Make sure that you use options.UseChangeTrackingProxies() in your DbContext settings.
```

# [C# (XPO)](#tab/tabid-csharp-xpo)

```csharp
[RuleValueComparison(
    null, DefaultContexts.Save, ValueComparisonType.NotEquals, Priority.Unspecified)]
public Priority Priority {
    get { return priority; }
    set {
        SetPropertyValue(nameof(Priority), ref priority, value);
    }
}
// ...
public enum Priority {
    Unspecified,
    Low,
    Normal,
    High
}
```

### Customize the Validation Rule

The rule generated by `RuleRequiredFieldAttribute` is loaded to the [Application Model](xref:112580)'s [](xref:DevExpress.ExpressApp.Validation.IModelValidationRules) node. You can use the [Model Editor](xref:112582) to customize this rule.

You can create new rules in the Model Editor as well. This is helpful when you need to add a rule to a property that is implemented in a base class from a third-party library.

For more information, refer to the following topic: [](xref:402980).

### Required Field Mark 

XAF appends an asterisk symbol (*) to an editor label if the corresponding field value is required.

You can customize the appended string in the [Model Editor](xref:112582). Use the [IModelLayoutManagerOptionsValidation.RequiredFieldMark](xref:DevExpress.ExpressApp.Validation.IModelLayoutManagerOptionsValidation.RequiredFieldMark) property to specify a custom string.
