syntax:
  content: public ParametrizedAction(Controller owner, string id, string category, Type valueType)
  parameters:
  - id: owner
    type: DevExpress.ExpressApp.Controller
    description: A [](xref:DevExpress.ExpressApp.Controller) object that represents a [Controller](xref:112621) to which the created Parametrized Action is added. This object is assigned to the [ActionBase.Controller](xref:DevExpress.ExpressApp.Actions.ActionBase.Controller) property.
  - id: id
    type: System.String
    description: A string value representing the created Parametrized Action's identifier. This value is assigned to the [ActionBase.Id](xref:DevExpress.ExpressApp.Actions.ActionBase.Id) property.
  - id: category
    type: System.String
    description: A string value that represents a category of the created Parametrized Action. This value is assigned to the [ActionBase.Category](xref:DevExpress.ExpressApp.Actions.ActionBase.Category) property and used to map this Action to the appropriate [Action Container](xref:112610).
  - id: valueType
    type: System.Type
    description: A [](xref:System.Type) value that specifies the type of the created Parametrized Action's [ParametrizedAction.Value](xref:DevExpress.ExpressApp.Actions.ParametrizedAction.Value).
seealso: []

Via this method, a Parametrized Action is created and added to the [Controller](xref:112621) specified by the _owner_ parameter.

Usually, you do not need to create [](xref:DevExpress.ExpressApp.Actions.ParametrizedAction) objects manually. Instead, use the parent Controller's Designer to add Actions. For details, refer to the [Actions](xref:112622) topic.
