If [DetailView.UseAsyncLoading](xref:DevExpress.ExpressApp.DetailView.UseAsyncLoading) is set to **true**, note the following:

* when [IModelAsync.UseAsyncLoading](xref:DevExpress.ExpressApp.Model.IModelAsync.UseAsyncLoading) is **true**, the [DetailView.CurrentObject](xref:DevExpress.ExpressApp.DetailView.CurrentObject) property is set to _null_ in the **Controller.OnActivate** method;
* when you open a Detail View from the [Navigation](xref:113198), the [DetailViewCreatingEventArgs.Obj](xref:DevExpress.ExpressApp.DetailViewCreatingEventArgs.Obj) argument of the @DevExpress.ExpressApp.XafApplication.ViewCreating and @DevExpress.ExpressApp.XafApplication.DetailViewCreating events is set to *null*;
* when you open a Detail View from a List View (by double-clicking a record or using the **OpenObject** Action), the [DetailViewCreatingEventArgs.Obj](xref:DevExpress.ExpressApp.DetailViewCreatingEventArgs.Obj) argument of the @DevExpress.ExpressApp.XafApplication.ViewCreating and @DevExpress.ExpressApp.XafApplication.DetailViewCreating events is set to the object from the List View's Object Space;
* when [IModelAsync.UseAsyncLoading](xref:DevExpress.ExpressApp.Model.IModelAsync.UseAsyncLoading) is **true** and you use [DetailView.ObjectSpace](xref:DevExpress.ExpressApp.CompositeView.ObjectSpace) before the Detail View's object is initialized (for example, in **Controller.OnActivated**), an exception occurs. Move the logic that depends on this property to the @DevExpress.ExpressApp.View.CurrentObjectChanged handler.
