XAF's EF Core implementation does not support the following data model configurations: entities with composite/compound keys (multiple properties as an entity key), [keyless entity types](https://learn.microsoft.com/en-us/ef/core/modeling/keyless-entity-types), and [owned entity types](https://learn.microsoft.com/en-us/ef/core/modeling/owned-entities). In such scenarios, inherit your EF Core entity from XAF's `BaseObject` class or define your own class with a single @System.Guid, numeric (@System.Int32, @System.Int64 ), or @System.String primary key. You can also use [non-persistent objects](xref:116516) to integrate XAF UI with legacy systems.
