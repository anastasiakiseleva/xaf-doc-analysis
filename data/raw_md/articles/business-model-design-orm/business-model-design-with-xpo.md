# Business Model Design with XPO

In [XPO](https://www.devexpress.com/Products/NET/ORM/), a business model is declared by classes (Persistent Objects) that will define the database structure, and consequently, the user interface. Classes represent database tables and class properties refer to database table columns. You can declare persistent objects directly in code (Code First) or by using the XPO Data Model Designer (Model First), or generate code from an existing database (Database First). The topics in this section provide information on how to properly declare your data classes.

> [!NOTE]
> It is recommended that you first review the [XPO tutorials](xref:2263) to learn about the fundamental XPO concepts before creating an XPO-based XAF application.
